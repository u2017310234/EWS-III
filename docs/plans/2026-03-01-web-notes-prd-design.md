# PRD：Web 端本地优先笔记（对齐 Obsidian 常用核心体验）

- 版本：v0.1
- 日期：2026-03-01
- 语言：zh-CN
- 终端：桌面 + 手机 + 平板
- 浏览器：Chromium（完整本地文件夹读写）+ Firefox/Safari（导入/导出降级）
- 交付方式：一次性交付（本 PRD 范围内功能全部完成后验收）

## 1. 背景与目标

### 1.1 背景
Obsidian 的核心体验由本地 Vault、Markdown 编辑、双链与快速检索驱动。本项目旨在将“常用核心能力”迁移到网页端，并保持“本地优先”的数据所有权与使用习惯。

### 1.2 目标
- 在网页端提供可用的“本地优先” Markdown 知识库体验。
- Chromium 浏览器下可直接读写用户选择的本地文件夹（真实落盘）。
- Firefox/Safari 通过导入/导出实现可用闭环（导入 → 编辑 → 导出备份/迁移）。
- 覆盖经确认的 12 项常用核心功能（见第 3 章范围）。

### 1.3 非目标（明确不做）
- 不做插件系统、不做主题市场、不做 Obsidian 插件兼容。
- 不做 Sync / Publish、账号体系、协作编辑。
- 不做 PWA（可安装、离线缓存策略、后台同步等不在范围内）。
- 不追求 100% 还原桌面端所有细节与全部 Core plugins。

### 1.4 成功标准（验收导向）
- Chromium：用户选择本地文件夹后，创建/编辑/重命名/移动/删除文件均能实时落盘；刷新页面后仍可继续使用（权限丢失时有可理解的重授权流程）。
- Firefox/Safari：可导入文件或 zip 创建 Vault；可编辑与检索；可导出 zip 作为完整备份；再次导入可恢复结构与内容。
- 12 项功能均可用；移动端可完成“浏览文件树 → 打开 → 编辑 → 搜索 → 跳转 → 反链查看 → 标签筛选”的核心路径。

## 2. 方案与约束

### 2.1 推荐方案：纯前端 SPA + VaultAdapter 抽象（本地文件夹/IndexedDB/预留远端）
采用单页应用（SPA），以统一的 `VaultAdapter` 抽象文件系统能力：
- `FsAccessAdapter`：基于 File System Access API（Chromium），直接读写本地文件夹。
- `IndexedDbAdapter`：用于 Firefox/Safari 的降级存储（导入后写入 IndexedDB），通过导出 zip 迁移。
- `RemoteAdapter`：仅预留接口，不实现具体服务（未来可对接远端文件服务以扩展存储方式）。

### 2.2 关键约束
- 不引入强依赖后端服务才能使用的能力；远端接口仅作为适配器预留。
- 搜索与索引需在 Web Worker 中运行，避免阻塞 UI。
- 图谱与嵌入渲染需具备性能/循环保护与降级策略。

## 3. 范围（功能清单）

本 PRD 仅实现以下“常用核心子功能”：
1) Vault 打开/创建（本地文件夹：Chromium；FF/Safari：导入后在浏览器内管理）
2) 文件树：新建/重命名/移动/删除/拖拽排序
3) Markdown 编辑器：源码 + 预览（分屏/切换），基础快捷键
4) WikiLink：`[[链接]]`、`![[嵌入]]`、标题/块引用（`[[A#H2]]` / `[[A^block]]`）
5) 反链/提及（Backlinks）与未创建链接提示
6) 全局搜索（文件名/内容），结果高亮与跳转
7) 标签 `#tag`：聚合页/筛选（含 `#a/b`）
8) 图谱（Graph）：节点/边、筛选、点击跳转
9) 工作区布局：多窗格/标签页、左右分栏、可保存布局（移动端降级）
10) 命令面板（Command Palette）+ 可配置快捷键
11) 设置：主题（浅/深）、字体/字号、基本编辑偏好
12) 导入/导出：文件/zip（用于 FF/Safari 的降级与备份）

## 4. 信息架构与页面结构

### 4.1 App Shell（主要布局）
- 顶栏：当前 Vault 名称、全局搜索入口、命令面板入口、布局/设置入口
- 左侧栏：文件树（可收起；移动端为抽屉）
- 主区域：编辑器 Tabs + 分栏容器（桌面/平板）；移动端为单栏 + Tabs 切换
- 右侧栏：反链面板 / 标签面板（至少支持反链与标签两个面板）

### 4.2 独立视图（作为主区域 Tab）
- 图谱视图（Graph）
- 设置页（Settings）

## 5. 关键用户流程

### 5.1 首次进入与打开 Vault
- Chromium：
  1. 点击「打开本地文件夹」
  2. 浏览器授权目录访问
  3. 扫描目录与构建索引（显示进度）
  4. 进入工作区
- Firefox/Safari：
  1. 提示不支持直接读写本地文件夹
  2. 点击「导入文件/zip」
  3. 解析并写入浏览器内存储（IndexedDB）
  4. 构建索引后进入工作区

### 5.2 编辑与保存
1. 文件树点击打开笔记
2. 在编辑器中输入（自动保存，且提供手动保存命令）
3. 切换「编辑 / 预览 / 分屏」

### 5.3 双链创建与未创建链接
1. 在笔记中输入 `[[New Note]]`
2. 预览中点击该链接
3. 弹出「创建并打开」→ 生成对应文件并打开

### 5.4 搜索跳转
1. 顶栏搜索或快捷键打开搜索
2. 输入关键词
3. 结果列表回车打开并定位高亮

### 5.5 备份与迁移
- 任意浏览器可导出 zip。
- Firefox/Safari 依赖导出 zip 作为主要备份/迁移方式。

## 6. 功能需求与验收要点（FR）

### FR1 Vault 管理
- 能力：打开/关闭 Vault、最近打开列表、权限失效提示与重授权（Chromium）
- 验收：刷新页面后可从“最近 Vault”重开；权限失效有明确操作路径恢复

### FR2 文件树
- 能力：新建文件/文件夹、重命名、移动（拖拽/菜单）、删除（确认）
- 验收：操作后索引与 UI 同步更新；打开文件不会丢失

### FR3 Markdown 编辑器
- 能力：编辑/预览/分屏；撤销重做；查找替换；基础快捷键（保存/搜索/命令面板/模式切换）
- 验收：编辑稳定；移动端可用；保存后内容可在重新打开时保持一致

### FR4 Markdown 渲染（预览）
- 支持：标题、列表、引用、代码块、表格（GFM）、任务列表
- 支持内链渲染：`[[...]]` 可点击；`![[...]]` 可嵌入（带循环/深度保护）
- 支持锚点：`[[A#Heading]]`、`[[A^block]]`
- 验收：嵌入可用；循环嵌入不会卡死并给出提示

### FR5 WikiLink 解析与补全（最小可用）
- 支持：`[[name]]`、`[[name|alias]]`、`![[name]]`
- 未创建链接：可标记并提供创建入口
- 验收：点击未创建链接可创建文件；重命名后链接至少能被识别为断链并可提示修复

### FR6 反链（Backlinks）
- 展示：链接到当前笔记的引用列表（含上下文片段）
- 未创建链接提及：基于 `[[Name]]` 的未创建链接集合（不做 NLP）
- 验收：编辑任意文件新增/删除链接后，反链在可接受延迟内更新（目标 < 1s）

### FR7 全局搜索
- 范围：文件名 + 正文内容
- 体验：结果高亮、回车打开定位
- 验收：输入时不阻塞 UI（Worker）；结果与跳转准确

### FR8 标签
- 解析：`#tag`、`#a/b`；建议忽略代码块内标签
- 功能：标签列表、点击标签筛选笔记列表并可跳转
- 验收：新增/删除标签后列表更新正确

### FR9 图谱（Graph）
- 数据：节点=笔记、边= WikiLink
- 功能：缩放/拖拽、至少一种筛选（如按标签/文件夹）、点击打开笔记
- 验收：中等规模可交互；大图有性能提示/建议筛选

### FR10 工作区布局
- 桌面/平板：多 Tab、左右分栏（最小：左右拆分 + Tab）
- 移动端：单窗格 + Tab；侧栏为抽屉
- 持久化：每 Vault 保存布局
- 验收：刷新后布局恢复；不同 Vault 布局互不影响

### FR11 命令面板与快捷键
- 命令面板：模糊搜索、回车执行
- 快捷键：查看/修改、冲突检测
- 验收：可键盘完成打开搜索/切换预览/导出 zip 等关键动作

### FR12 设置
- 主题：浅色/深色
- 编辑偏好：字体/字号/行高/自动换行/Tab 宽度（最小集合即可）
- 验收：设置即时生效并持久化（明确全局或按 Vault 维度，产品需固定一种策略）

### FR13 导入/导出（zip）
- 导入：zip（含目录结构）与多文件导入（尽量保留层级）
- 导出：导出当前 Vault 为 zip（保留目录结构）
- 同名冲突：至少支持覆盖与跳过
- 验收：导出 zip 解压后结构正确；再次导入可恢复核心体验

## 7. 数据、索引与更新策略

### 7.1 索引类型
- 文件元数据：路径、标题（首个 H1 或文件名）、mtime、大小
- 链接索引：出链/入链、未创建链接集合
- 标签索引：tag → 文件集合
- 搜索索引：全文索引（Worker 内构建与查询）

### 7.2 更新策略
- 单文件保存：增量更新索引（反链/标签/搜索）
- 大批量导入或首次打开：显示进度并可取消；完成后进入“索引就绪”状态

## 8. 兼容与降级策略
- Chromium：使用 File System Access API 读写真实文件夹；权限失效需可重授权。
- Firefox/Safari：不承诺直接读写本地文件夹；依赖导入/导出闭环；数据存于 IndexedDB。
- 移动端：不强求多分栏拖拽；确保核心编辑与检索路径可用；图谱可默认关闭或简化。

## 9. 非功能需求（NFR）
- 性能：索引/搜索不阻塞 UI；大操作有进度；图谱/嵌入有降级与保护
- 可靠性：自动保存；异常中断不丢“已保存/已写入”的内容
- 隐私：默认不上传任何笔记内容；远端适配器仅预留不启用
- 可用性：键盘可达（命令面板/搜索/Tab 切换）；基础无障碍（焦点与 aria 标签）

## 10. 预留远端文件交互接口（不实现）

### 10.1 适配器抽象
前端定义统一 `VaultAdapter`（或同等抽象），覆盖：
- 列目录、读文件、写文件、移动/重命名、删除
- 批量导入/导出（建议以 zip 形式）

### 10.2 预留实现
- `RemoteAdapter`：面向未来远端文件服务（HTTP API）。本期不提供服务端实现与鉴权方案。
